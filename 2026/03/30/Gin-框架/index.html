RESTful API

REST是一种软件架构风格->翻译为表征状态转移

REST的含义就是客户端与Web服务器之间进行交互的时候，使用HTTP协议中的4个请求方法代表不同的动作。

  • GET用来获取资源
  • POST用来新建资源
  • PUT用来更新资源
  • DELETE用来删除资源。

参数绑定

.ShouldBind()能够基于请求的Content-Type自动提取JSON、form表单和QueryString类型的数据，并把值绑定到指定的结构体对象。

Gin 拿到结构体后，并不会马上填数据，而是先通过反射查看结构体的类型信息（Type），特别是字段上的 Tag。

如：对于 User string json:"user" 这个字段：

  • 反射读取到字段名为 User。
  • 反射读取到 Tag 为 json:"user"。
  • 映射建立：Gin 记下，如果前端传入的 JSON 里有 user 这个键，就把值给 User 字段。

中间件

Gin框架允许开发者在处理请求的过程中，加入用户自己的钩子（Hook）函数。这个钩子函数就叫中间件，中间件适合处理一些公共的业务逻辑，比如登录认证、权限校验、数据分页、记录日志、耗时统计等。

可以理解为大家共用的中间函数，一次写定，多次复用。

定义中间件

Gin中的中间件必须是一个gin.HandlerFunc类型。

保所有中间件都能接收 *gin.Context（上下文），这样才能在中间件之间传递数据和控制流程。

两种定义方式：

  • 直接定义函数

  • 采用闭包函数（最常用）

注册中间件

  • 全局路由注册

  • 单个路由注册

  • 路由组注册

跨域中间件cors

用于解决前后端分离架构下的跨域请求问题。

当前端应用（如 http://localhost:3000）需要向一个不同源的后端API（如 http://localhost:8080）请求数据时，就构成了跨域请求。默认情况下，浏览器会拦截。CORS则是为了放行的。

注意事项

当在中间件或handler中启动新的goroutine时，不能使用原始的上下文（c *gin.Context），必须使用其只读副本（c.Copy()）。

因为 Gin 为了高性能复用了 Context 对象，请求结束后原始的 Context 会被重置并归还给池子供其他请求使用，导致后台 Goroutine 拿到的指针可能指向了新请求的数据而非原来的数据，从而引发严重的数据竞争。
