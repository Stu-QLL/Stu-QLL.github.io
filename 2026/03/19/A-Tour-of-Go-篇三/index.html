并发

Go程

Go 程（goroutine）是由 Go 运行时管理的轻量级线程。

go f(x, y, z)

会启动一个新的 Go 程并执行

f(x, y, z)

注意：go 关键字后面的函数参数，是在**“启动新线程的那一瞬间”就计算好并复制过去**的，而不是等新线程运行时再去读变量的值。

go的并发哲学

不要通过共享内存来通信，而要通过通信来共享内存；通道是同步信号（只要从通道里获得结果，就证明对方一定任务结束了，而不需要不断的询问）

go并发的意义

如果切片非常大（比如有几亿个数字），或者计算逻辑非常复杂（比如加密、图像处理），单线程计算会很慢。通过启动两个 goroutine，这两个任务可以同时在不同的 CPU 核心上运行。相当于并发执行：总耗时 ≈ Max(任务A耗时, 任务B耗时)。如果任务很重，并发版会快接近一倍。

go并发的核心思维

通过关注数据的流动（通信），来自动管理过程的同步

维度 传统思维 (共享内存) Go 思维 (通信)
关注点 状态 (State)：变量现在是几？锁住了吗？ 行为 (Behavior)：数据在怎么流动？
控制方式 命令式：我要让线程 A 停，让线程 B 走。 声明式/流式：数据到了这里，自然就触发下一步。
同步手段 显式的锁 (Lock/Unlock)、条件变量。 隐式的通道阻塞 (<- 和 ->)。
出错风险 死锁、竞态条件、忘记解锁。 死锁（管道没人读/写）、协程泄漏（忘了发数据）。
比喻 交警指挥每一辆车。 设计好单行道和环岛，车流自动运行。

select 语句

使一个 Go 程可以等待多个通信操作。

单流程下一个go只能监控一个channel的状态，select可以实现监控多个channel的状态

注意：在 select 语句中，case 后面的表达式代表的是：“我想做这个动作，现在能做吗？”

比如case c <- x

它不是问：“c <- x 这个表达式成立吗？”
它是问：“我现在可以向 c 发送 x 吗？如果可以（即：有另一个协程正在从 c 读数据，或者 c 是有缓冲且未满的），我就立刻执行发送，然后运行后面的代码块。”

当 select 中的其它分支都没有准备好时，default 分支就会执行。

sync.Mutex

Mutual Exclusion Lock（互斥锁）

当多个 Goroutine 同时访问和修改共享资源（如全局变量、共享的结构体字段、切片、Map 等）时，如果没有保护机制，就会导致数据竞争 (Data Race)

sync.Mutex 的作用就是：在任何时刻，只允许一个 Goroutine 持有锁并访问共享资源，其他试图获取锁的 Goroutine 必须等待。
