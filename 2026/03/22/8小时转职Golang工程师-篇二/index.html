多态

Go 没有 类继承，因此不支持基于“父类引用指向子类对象”的多态。
Go 的多态完全基于 接口（Interface） 和 方法集（Method Set）。只要一个类型实现了某个接口的所有方法，它就可以被视为该接口类型，从而实现多态。

什么是多态？

简单来说：同一个接口，多种实现

定义一个函数，它接收一个接口类型作为参数。在运行时，你可以传入任何实现了该接口的具体类型，程序会根据传入的具体类型执行对应的逻辑。

基本要素

  • 接口定义行为：定义一组方法签名
  • 隐式实现：类型无需声明 implements，只要自然拥有这些方法，就自动实现了接口
  • 口变量持有具体值：接口类型的变量可以存储任何实现了该接口的具体类型的值

关键特性

  • 隐式实现（implicit Implementation)

  • 鸭子类型（Duck Typing）

    “如果它走起来像鸭子，叫起来像鸭子，那它就是鸭子。”
    在 Go 中，如果一个类型拥有接口所需的所有方法，它就是该接口类型。编译器在编译期就会检查这一点，所以它是静态类型安全的，而不是动态语言的运行时检查。

  • 空接口interface{}

    如果一个接口没有任何方法 (interface{})，那么所有类型都实现了它。

接口冲突问题

情况一：两个接口有相同的方法签名（完全兼容）

两个接口定义了完全相同的方法，一个类型只要实现了一次该方法，就同时实现了这两个接口。

情况二：两个接口有相同的方法名，但签名不同（编译错误）

情况三：结构体嵌入导致的“方法提升”冲突

解决方法：采用显式调用

reflection

反射是指一类应用，它们能够自描述和自控制。也就是说，这类应用通过采用某种机制来实现对自己行为的描述（self-representation）和监测（examination），并能根据自身行为的状态和结果，调整或修改应用所描述行为的状态和相关的语义。

具体来说，反射（Reflection） 是指程序在运行时检查自身结构和行为的能力。它允许你在运行时获取变量的类型信息、值信息，甚至修改变量的值。

变量的内置 pair 结构

  • 接口变量含有一个对应的pair内存结构，包括（type, value）两部分
  • type 包括 static type和concrete type.
    • Static Type (静态类型)：这是编译器看到的类型。当声明一个变量时，它的类型就固定了。
      • 例如：var r io.Reader，这里的 io.Reader 就是静态类型。
    • Concrete Type (具体类型/动态类型)：这是变量在运行时实际存储的值的类型。
      • 只有当变量是接口类型时，这个概念才有意义。接口变量像一个盒子，盒子上贴着标签（静态类型，如 io.Reader），但盒子里装的东西可以是任何实现了该接口的具体类型（如 *os.File, *bytes.Buffer, *strings.Reader）。
      • 这个“盒子里装的东西”的类型，就是 Concrete Type。
  • 类型断言能否成功，取决于变量的concrete type，而不是static type. 因此，一个 reader变量如果它的concrete type也实现了write方法的话，它也可以被类型断言为writer.
    • 机制：当进行类型断言时，Go 运行时系统会检查接口变量内部存储的 Concrete Type (具体类型) 是否实现了目标接口（或是否等于目标类型）。
    • 无关性：它完全不关心该接口变量的 Static Type (静态类型) 是什么
