切片

切片在传参上类似于引用传递

slice虽然传了数组指针，但是它本质还是值拷贝。slice内包含了数组指针，传递slice相当于复制了数组指针。

切片本质上是一个包含三个字段的结构体：

  • 修改元素内容：因为两个结构体里的 Data 指针指向同一块底层数组内存，所以通过新切片修改数组元素（如 slice[0] = 99），原切片看到的也会变。

    相似的, map 也是类似于引用传递，在一个函数中修改形参map，会相应的修改原有的实参map

  • 修改长度/容量：如果在函数内部对切片进行 append 操作扩容，或使用 slice = slice[:1] 改变长度，这只是修改了函数内部那个副本的 Len、Cap 或 Data 指针（如果扩容发生了重分配）。原切片的 Len、Cap 和 Data 不会改变。

copy函数

其可以将底层数组的slice一起拷贝

切片和底层数组的指针指向同一地址，因此修改切片的值，具有同一个底层数组的切片们都会被修改。

而 copy 则会生成一个全新的底层数组以脱离切片对原数组的依赖。
