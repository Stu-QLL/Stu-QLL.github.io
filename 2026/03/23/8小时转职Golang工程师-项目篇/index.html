典型 TCP 服务器启动与运行的生命周期流程

1. socket listen (创建监听套接字)

  • 含义：这是服务器初始化的第一步。**Socket （套接字）**是应用层与传输层（TCP/UDP）之间通信的中间软件抽象层。
  • 具体操作
    • 创建 Socket：调用系统创建一个网络端点。
    • 绑定地址：调用 bind() 将 socket 绑定到特定的 IP 地址和端口号。
    • 开始监听：调用 listen() 将该 socket 转换为被动模式，准备接收传入的连接请求。
  • 状态：此时服务器已经开门，但未处理任何客户，只是在等待。

2. accept (接受连接)

  • 含义：这是服务器进入等待客户端连接的关键步骤。
  • 具体操作
    • 调用 accept() 函数。
    • 阻塞特性：在标准的简单实现中，这是一个阻塞调用。如果没有客户端连接，程序会停在这里一直等待。
    • 建立连接：当有客户端发起连接请求（三次握手完成）后，accept 会从内核的连接队列中取出一个连接，并返回一个新的 Socket 文件描述符（通常称为 conn 或 clientSocket）。
    • 注意：原来的监听 socket (listen socket) 继续留在原地监听新的连接，而新生成的 socket 专门用于与当前这个特定的客户端通信。

3. do handle (处理业务逻辑)

  • 含义：与客户端进行实际的数据交互。
  • 具体操作
    • 使用上一步生成的新 socket 进行 read (读取请求) 和 write (发送响应)。
    • 并发模型
      • 多线程/协程模型 (Go 常用)：在实际的 Go 服务器中，这一步通常会启动一个新的 Goroutine (go handle(conn))。这样主循环可以立即跳回 accept 去接收下一个客户，实现高并发。
    • 业务内容：解析协议（如 HTTP、RPC）、查询数据库、计算结果、返回数据等。

4. close listen socket (关闭监听套接字)

  • 含义：服务器停止服务，释放资源。
  • 具体操作
    • 调用 close() 关闭最初创建的监听 socket。
    • 后果
      1. 端口被释放，其他程序可以绑定该端口。
      2. 不再接受新的连接请求。
      3. 已经在处理的连接（如果在 do handle 中生成的独立 socket 没有被同时关闭）可能还会继续完成剩余的数据传输，或者根据实现方式强制断开。
